3.Processo
3.1 Defina a tarefa e os critérios de sucesso
 a) Desempenho e precisão: Quão bem o modelo precisa executar na tarefa? ex: a resposta pode ser genérica
 b) Latência: Qual é o tempo de resposta aceitável para o modelo? Isso dependerá dos requisitos em tempo real do seu app e das expectativas do usuário.
	ex: Para chatbot uma resposta mais prática. Sacrificar um pouco da qualidade para ter uma resposta mais instantânea
 c) Preço: Qual é o orçamento para executar o modelo? Considere fatores como o custo por chamada de API, o tamnho do modelo e a frequência de uso.

3.2 Desenvolva casos de teste - como saber que esta gerando o que espero
3.3 Escreva o prompt incial
3.4 Teste-o contra exemplos
3.5 Refine o prompt conforme os resultados
3.6 Ponha-o em produção

4.Técnicas Básicas
MARKDOWN - Markdown Syntax é uma sintaxe usada para padronizar e facilitar formatação de texto na web, utilizada em aplicativos como Slack e GitHub. Textos estilizados com Markdown são, na maioria dos casos, apenas texto com caracteres não-alfabéticos, como #, \* e ![](), usados para a configuração de títulos, listas, itálico, negrito e inserção de imagens.

Exemplo:
# Título <h1>
## Título <h2>
### Título <h3>
#### Título <h4>
##### Título <h5>
###### Título <h6>
